# Task Prompt

你是一家小型软件公司的 AI 方案顾问。团队计划在未来 8 周内上线一个“内部研究助理型 agent”，当前有三个可选推进方案：

## 方案 A：快速上线
- 1 周内可启动
- 依赖现成工作流和通用 skill
- 前期效果最容易展示
- 定制空间有限
- 数据权限控制一般

## 方案 B：深度定制
- 需要 4 周打底
- 可以接入内部知识库和更细粒度权限
- 长期潜力最高
- 前期交付慢
- 对团队工程能力要求最高

## 方案 C：折中推进
- 2 周可启动
- 可以先做通用版本，再逐步接入内部资料
- 成本和复杂度中等
- 体验较稳
- 后续扩展性不如方案 B

## 团队情况
- 技术团队只有 2 人
- 管理层希望 2 周内看到可演示成果
- 公司对内部资料权限较敏感
- 预算中等，不追求最低成本
- 未来 3 个月内很可能要接入内部文档和历史项目资料

请写一份**选型建议 memo**，回答以下内容：
1. 你推荐哪个方案
2. 为什么不推荐另外两个方案
3. 未来 8 周的落地顺序建议
4. 主要风险与缓解措施

# Required Output Format

必须严格使用以下结构输出：

# 选型建议
## 推荐结论
...
## 不推荐其他方案的原因
...
## 8周落地顺序
...
## 风险与缓解
...

# Constraints

- 使用中文
- 总长度控制在 400–700 字
- 必须给出明确推荐，不能只做优缺点罗列
- 必须结合题目中的团队约束进行判断
- 不允许引入题目之外的假设性事实来替代分析

# Deliverables

最终交付为一份结构化 memo，包含：
- 明确推荐方案
- 对其他方案的否定理由
- 8 周落地顺序
- 风险与缓解措施
